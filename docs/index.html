Cold outreach that doesn't
make things up.

Холодний аутріч,
який нічого не вигадує.

LeadFlow AI is an agentic TypeScript pipeline that enriches B2B leads from their websites and writes personalized cold-outreach copy — opener, ice-breaker, subject line, and an honest fit score. Anti-hallucination by construction, not by promise.

LeadFlow AI — це агентний TypeScript-пайплайн, що збагачує B2B-ліди з їхніх сайтів і пише персоналізовані тексти холодного аутрічу — opener, ice-breaker, тему листа та чесну оцінку fit. Анти-галюцинації за конструкцією, а не за обіцянкою.

Built like production, not a demo

Зроблено як продакшен, а не демо

Six design choices that separate this from a 20-line "wrap a LLM" script.

Шість дизайн-рішень, що відрізняють це від 20-рядкового "wrap a LLM" скрипта.

⊘

Anti-hallucination by construction

Анти-галюцинації за конструкцією

The LLM only sees text that the code actually scraped. System prompt forbids invented facts. Thin context → fit_score ≤ 2 automatically.

LLM бачить лише текст, який код реально витягнув. System prompt забороняє вигадані факти. Тонкий контекст → fit_score ≤ 2 автоматично.

{ }

One zod schema, two providers

Одна zod-схема, два провайдери

Anthropic tool-use (with prompt caching) and Groq json_object both converge on the same zod validator. Switch with one env var.

Anthropic tool-use (з prompt-кешем) і Groq json_object сходяться на тому самому zod-валідаторі. Перемикання через одну env-змінну.

⟲

Resilient batch processing

Стійка обробка батчу

Bad fetch · LLM rate-limit · malformed JSON — none of them drop the batch. The lead gets a fallback template, the run continues.

Битий fetch · rate-limit LLM · поламаний JSON — жодне з цього не вбиває батч. Лід отримує fallback-шаблон, прогін триває.

⤓

Idempotent & cached

Ідемпотентність & кеш

Per-domain JSON cache means the second run touches no network. Already-processed leads are skipped without --force.

Per-domain JSON-кеш — повторний прогін не лізе в мережу. Вже оброблені ліди скіпаються без --force.

⇄

Polite concurrency

Ввічлива конкурентність

Tiny custom pLimit (no dependency) so you don't blast a site or melt the LLM rate-limit. Configurable.

Маленький власний pLimit (без залежностей), щоб не задовбати сайт і не вийти за rate-limit LLM. Конфігурується.

▶

Offline demo mode

Офлайн демо-режим

--mock reads data/fixtures/<domain>.txt instead of fetching. CI runs deterministically, screenshots reproduce, no secrets required.

--mock читає data/fixtures/<domain>.txt замість fetch'у. CI прогоняється детерміновано, скріншоти відтворюються, секрети не потрібні.

How it works

Як працює

Deterministic data collection, one constrained LLM call, single zod schema, fallback that the model can't bypass.

Детермінований збір даних, один обмежений LLM-виклик, єдина zod-схема, fallback, який модель не може обійти.

1

Fetch leads

Завантажити ліди

CSV or Google Sheets, behind one LeadsSource interface.

CSV або Google Sheets за одним LeadsSource-інтерфейсом.

2

Enrich (fetch site)

Збагатити (тягнемо сайт)

Timeout · UA · content-type guard. Title, meta, H1/H2, body to ~4000 chars + signal detection.

Timeout · UA · перевірка content-type. Title, meta, H1/H2, тіло до ~4000 символів + сигнали.

3

LLM personalize

LLM персоналізує

One structured call. Claude tool_choice or Groq json_object. Cache_control on system prompt.

Один структурований виклик. Claude tool_choice або Groq json_object. Cache_control на system prompt.

4

Validate (zod)

Валідація (zod)

Any deviation from schema → fallback template. Batch never drops.

Будь-яке відхилення від схеми → fallback. Батч не падає.

5

Write output

Запис виводу

CSV with BOM (Excel/Numbers-safe), optional Sheets append, optional Resend smoke-test email.

CSV з BOM (для Excel/Numbers), опційно Sheets append, опційно тестовий лист через Resend.
